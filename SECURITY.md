# 安全说明

本项目当前实现的是代码层面的基础抗刷和安全防护，目标是降低小规模恶意请求、重复提交、接口滥用和 AI 成本被刷的风险。

请注意：代码不能完全防住大流量 DDoS。真正的大流量攻击需要接入 Cloudflare、腾讯云高防、阿里云高防、WAF、CDN、高防 IP、服务器防火墙和 Nginx 限流等基础设施。

## 已实现

- Helmet 安全响应头。
- CORS 白名单，默认只允许本地前端来源。
- 登录接口限流：单 IP 每分钟 5 次。
- 管理员登录接口限流：单 IP 每分钟 5 次。
- AI 分析接口限流：单用户每分钟 3 次，单 IP 每分钟 10 次。
- 支付创建订单限流：单用户每分钟 3 次。
- 全局接口限流。
- IP 黑名单：后台可封禁和解除封禁。
- 临时封禁：高频请求超过阈值后自动临时封禁。
- 请求日志：记录 IP、路径、方法、User-Agent、状态码、耗时、用户 ID、是否限流、是否黑名单拦截。
- 安全事件：记录限流、黑名单、疑似攻击、高频请求、404、5xx 等事件。
- AI 分析防重复提交：同一用户短时间内提交相同内容会复用上一次成功结果。
- 支付订单防重复提交：同一用户同一商品 10 分钟内只保留一个待支付订单。
- 支付回调幂等：已支付订单重复回调不会重复发放会员或积分。
- 支付回调验签和金额校验。
- 请求体大小限制。

## 正式上线前必须配置

- 域名解析。
- CDN / WAF。
- 高防策略。
- 服务器防火墙。
- Nginx 限流和连接数限制。
- 后端限流。
- 日志监控和告警。
- 支付密钥、AI Key、JWT 密钥、数据库密码放入环境变量。
- GitHub 仓库必须排除 .env、node_modules、日志文件、上传临时文件、构建缓存。

## 后续建议

- 多实例部署时，将当前进程内限流迁移到 Redis。现在的代码已经按 IP 和用户维度做了限流，但进程内 bucket 不适合多台服务器横向扩容。
- 接入云厂商安全日志后，把 Cloudflare、Nginx、宝塔、腾讯云、阿里云的安全事件同步到 security_events。
- 为支付回调增加第三方平台证书验签逻辑。
- 为上传图片增加独立对象存储、病毒扫描和内容安全审核。
